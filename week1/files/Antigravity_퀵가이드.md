# Antigravity 퀵가이드

> 처음 사용하는 분을 위한 5분 안내서

---

## Antigravity가 뭔가요?

Google이 만든 무료 AI 코딩 도구입니다.
채팅으로 "이런 앱 만들어줘"라고 말하면 AI가 코드를 짜고, 미리보기까지 보여줍니다.
코딩을 몰라도 됩니다.

---

## 시작하기

### 1단계: 접속

1. 브라우저에서 **idx.google.com** 접속
2. Google 계정으로 로그인

### 2단계: 새 프로젝트 만들기

1. "New Project" 클릭
2. 프로젝트 이름 입력 (예: "대출계산기")
3. 템플릿에서 **"Next.js"** 선택
4. "Create" 클릭
5. 잠시 기다리기 (30초~1분)

### 3단계: AI에게 말하기

1. 화면 왼쪽 또는 하단의 **AI 채팅창** 찾기
2. 기준서(또는 프롬프트)를 **통째로 복사해서 붙여넣기**
3. Enter 누르고 기다리기 (2~5분)

### 4단계: 결과 확인

1. 화면 오른쪽의 **"Preview"** 탭 클릭
2. 만들어진 앱이 바로 보입니다

---

## 자주 만나는 상황

| 상황 | 해결법 |
|------|--------|
| "Allow" 팝업이 뜸 | "Allow" 클릭. AI가 파일을 수정하는 정상 과정입니다 |
| 화면이 하얗게 나옴 | AI 채팅에 "미리보기가 빈 화면이야. 확인해줘" 입력 |
| 빨간 에러 메시지 | 에러 메시지를 통째로 복사해서 AI 채팅에 붙여넣기 |
| "Module not found" | AI에게 "외부 패키지 없이 다시 만들어줘" 입력 |
| 진행이 멈춤 | AI에게 "계속 진행해줘" 입력 |
| 뭘 해야 할지 모름 | AI에게 "지금 뭘 해야 돼?" 입력 |
| 전부 꼬였을 때 | 새 프로젝트 만들고 기준서 다시 붙여넣기 |

### 결과가 이상할 때 — 원인부터 찾기

무작정 "고쳐줘"보다 **왜 이상한지** 먼저 물어보면 더 잘 고쳐집니다:

```
AI에게: "이 문제의 원인이 뭔지 분석해줘. 가능한 원인을 목록으로 보여줘."
```

기준서가 있다면 더 효과적:
```
AI에게: "기준서에는 이렇게 되어 있는데 실제로는 이렇게 동작해. 비교해줘."
```

## 수정하고 싶을 때

AI 채팅에 자연어로 말하면 됩니다:

- "색상을 좀 더 밝게 해줘"
- "글씨가 너무 작아, 키워줘"
- "이 기능이 빠져있어, 추가해줘"
- "모바일에서 보면 깨져, 고쳐줘"
- "입력 순서를 바꿔줘"

---

## 인터넷에 공개하고 싶을 때

AI에게 이렇게 말하세요:

```
이걸 다른 사람도 볼 수 있게 인터넷에 올리고 싶어. 방법 알려줘.
```

AI가 안내하는 대로 따라하면 됩니다.

---

## 핵심 원칙

1. **모르면 AI한테 물어보세요** - 에러든, 방법이든, 뭐든
2. **AI가 못 고치면 새로 시작하세요** - 새 프로젝트 만들고 기준서 다시 붙여넣기
3. **기준서가 좋으면 결과가 좋습니다** - AI한테 뭘 줬느냐가 결과를 결정합니다
