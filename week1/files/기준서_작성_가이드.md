# AI 프로젝트 기준서 작성 가이드 v2.0

> 이 문서는 두 파트로 구성되어 있습니다.
> - **Part 1**: 해설서 -- 왜 이런 단계를 거치는지, 각 단계에서 뭘 하는지 이해하기
> - **Part 2**: AI용 실행 문서 -- AI에게 그대로 복붙하면 AI가 알아서 진행하는 문서
>
> Part 1을 읽고 이해한 뒤, Part 2를 AI에게 복붙하면 됩니다.
> 집에서 혼자 해도 됩니다.

---

# Part 1: 해설서

## 왜 기준서가 필요한가

AI에게 "대출 계산기 만들어줘" 한 줄만 주면 -- 뭔가 나오긴 하지만 부족합니다.
같은 AI에게 기준서를 주면 -- 디자인, 기능, 계산 정확도까지 갖춘 결과가 나옵니다.

**같은 AI인데 결과가 다른 이유**: 뭘 줬느냐의 차이.

기준서 = AI에게 "이렇게 만들어"라고 전달하는 설계도.
사람한테 일 시킬 때도 "알아서 해"보다 "이런 식으로 해줘"가 결과가 좋은 것과 같습니다.

---

## 이 가이드의 핵심 원리 3가지

이 가이드는 "그냥 상식적으로 만든 단계"가 아닙니다.
검증된 3가지 원리를 따릅니다.

### 1. 인식 맞추기

AI와 내가 같은 그림을 보고 있는지 확인하는 과정.
AI가 내 말을 잘못 이해한 채로 쭉 가면, 나중에 전부 다시 해야 합니다.

**적용**: 매 단계가 끝날 때마다 AI가 "제가 이렇게 이해했는데 맞나요?" 하고 확인합니다.

### 2. 점진적 정제

"다 만들어줘" 한번에 시키면 AI가 대충 만듭니다.
단계를 나눠서 하나씩 쌓아가면, AI가 점점 더 정확하게 이해합니다.

**적용**: 7단계로 나눠서 정보 수집 -> 검증 -> 문서화 -> 실행 순으로 진행합니다.

### 3. 쌓아가기

기준서를 "다시 만들어줘"가 아니라 "여기만 고쳐줘"로 업데이트합니다.
전체를 다시 만들면 이미 좋았던 부분도 바뀔 수 있습니다.

**적용**: 수정할 때 기존 내용은 보존하고, 바뀐 부분만 추가/변경합니다.

---

## 7단계 전체 흐름

```
Phase 1: "뭘 만들 거야?"      -- AI와 같은 그림 맞추기
Phase 2: "뭘 알아야 해?"      -- AI가 맞춤 질문, 사용자가 답변
Phase 3: "놓친 거 없어?"      -- AI가 빈틈 잡기, 2차 점검
Phase 4: "기준서로 정리!"     -- 설계도 완성
Phase 5: "부족한 자료 수집"    -- AI가 알아서 찾기
Phase 6: "어떻게 만들지?"     -- 구조안 비교 후 선택
Phase 7: "복붙 -> 만들기"     -- 기준서 넣고 실행 + 점검
```

---

## 각 단계 해설

### Phase 1: 프로젝트 파악 -- "뭘 만들 거야?"

AI에게 만들고 싶은 걸 자유롭게 설명합니다. 한 줄이어도 됩니다.
AI가 듣고 정리해서 "제가 이렇게 이해했는데 맞나요?" 하고 확인합니다.

**왜 필요한가**: AI가 내 말을 잘못 이해한 채로 쭉 가면, 나중에 전부 다시 해야 합니다.
처음에 30초 확인하면 나중에 30분을 아낍니다.

**주제를 못 정겠다면** 이런 걸 떠올려 보세요:
- 엑셀로 반복하는 작업이 있나?
- 매번 계산기 두드리는 게 있나?
- 팀원/고객한테 보여줄 양식을 손으로 만들고 있나?

| 주제 예시 | 누구한테 필요한가 |
|-----------|-------------------|
| 대출이자 계산기 | 대출 비교할 때 |
| 견적서 생성기 | 프리랜서, 자영업자 |
| 식당 메뉴판 웹페이지 | 식당 사장님 |
| 운동 기록 트래커 | 운동하는 사람 |
| 회의록 템플릿 생성기 | 사무직 |
| 재고 관리 대시보드 | 소규모 매장 |
| 포트폴리오 사이트 | 취준생, 프리랜서 |

---

### Phase 2: 맞춤 질문지 -- "뭘 알아야 잘 만들지?"

AI가 프로젝트에 맞는 질문을 자동으로 만들어서 물어봅니다.
사용자는 답변만 하면 됩니다. **모르는 건 "모르겠어"라고 해도 됩니다.** AI가 대신 조사합니다.

AI가 묻는 5개 영역:

| 영역 | 묻는 것 |
|------|---------|
| A. 결과물 정의 | 최종 형태, 성공 기준, 참고 서비스 |
| B. 대상 분석 | 누가 쓰는지, 그 사람이 원하는 것, 실망할 것 |
| C. 재료 점검 | 이미 가진 자료, 부족한 자료 |
| D. 제약 조건 | 시간, 예산, 반드시 포함, 절대 금지 |
| E. 기존 사례 | 비슷한 서비스가 어떻게 되어 있는지 |

각 질문마다 AI가 "이 질문에 답하려면 이걸 준비하세요", "이건 여기서 찾을 수 있어요"도 같이 알려줍니다.

**예시:**
```
AI: "이 계산기의 성공 기준이 뭔가요?"
나: "계산 결과가 은행 사이트랑 같으면 성공이야"

AI: "비슷한 서비스 중 참고할 만한 게 있나요?"
나: "모르겠어"
AI: "네이버 대출 계산기, 토스 대출 비교 등을 찾아봤습니다. 참고하시겠어요?"
```

---

### Phase 3: 빈틈 잡기 -- "놓친 거 없어?"

Phase 2의 답변에서 빈 곳, 모순, 얕은 부분을 AI가 잡아냅니다.

**왜 필요한가**: 처음 만든 목록은 "내가 이미 아는 것"만 들어가기 쉽습니다.
AI에게 "내가 놓친 게 뭐야?"라고 물으면, 내가 생각 못한 관점을 찾아줍니다.

AI가 점검하는 7가지:
1. 답변이 모호한 부분
2. "이게 정말 필요한가?" 검증
3. 서로 모순되는 부분
4. 비슷한 서비스와 비교해서 부족한 점
5. 나중에 문제 될 수 있는 리스크
6. 결과물 구조에 대한 대안 비교
7. 놓치고 있을 수 있는 것

점검이 끝나면 AI가 핵심을 요약하고, 내가 맞는지 확인합니다.

---

### Phase 4: 기준서 작성 -- "설계도 완성"

Phase 2~3에서 쌓아온 모든 정보를 하나의 문서로 통합합니다.
**이 문서만 다른 AI에게 주면 바로 만들 수 있을 정도로** 구체적이어야 합니다.

기준서에 들어가는 항목:

| 항목 | 설명 |
|------|------|
| 프로젝트 요약 | 이름, 목적, 대상 사용자 |
| 필수 기능 목록 | 우선순위 포함 (필수/중요/선택) |
| 동작 규칙 | 계산식, 처리 흐름, 소수점 처리 등 |
| 화면 구조 | 레이아웃, 입력/결과 배치 |
| 디자인 방향 | 스타일, 색상, 분위기 (구체적으로) |
| 제약사항 | 하면 안 되는 것 |
| 확정된 결정사항 | Phase 2~3에서 정한 것들 요약 |

AI가 기준서를 만들어주면 아래를 확인하세요:

- [ ] 프로젝트 이름과 목적이 명확한가?
- [ ] 기능이 숫자로 나열되어 있는가?
- [ ] 계산식이 있으면 정확한 공식이 적혀 있는가?
- [ ] 화면 구조가 설명되어 있는가?
- [ ] 디자인 방향이 구체적인가? ("예쁘게"가 아니라 구체적 스타일)
- [ ] 하면 안 되는 것이 명시되어 있는가?

수정할 때는 "다시 만들어줘"가 아니라 **"여기만 고쳐줘"**로. 이미 좋은 부분까지 바뀌는 걸 막습니다.

---

### Phase 5: 자료 수집 -- "부족한 건 AI가 찾는다"

기준서를 기반으로 AI가 자동으로 수행합니다:

1. **Gap 분석** -- 기준서를 달성하려면 필요한데 아직 없는 자료 목록을 뽑는다
2. **직접 수집** -- 웹 검색, 공식 문서, 사례 조사 등으로 찾을 수 있는 것은 AI가 직접 수집한다
3. **사용자 요청** -- AI가 못 찾는 것 (개인 파일, 내부 자료)은 사용자에게 요청한다
4. **정리** -- 수집된 자료를 기준서 항목별로 매핑해서 정리한다

사용자는 AI가 요청하는 자료만 제공하면 됩니다.

**왜 필요한가**: 이 단계를 건너뛰면 AI가 근거 없이 짐작으로 만듭니다.
예를 들어 대출 계산기에서 정확한 이자 공식 없이 AI가 "아마 이렇겠지"로 계산 로직을 짜면, 결과가 은행과 다르게 나옵니다.

---

### Phase 6: 구조 설계 -- "어떻게 만들지?"

바로 만들기에 들어가지 않고, AI가 먼저 구조안 2~3가지를 비교해서 보여줍니다.
사용자가 골라주면 그 구조를 기반으로 상세 플랜을 작성합니다.

**6-1. 전체 구조 비교**
- AI가 큰 덩어리 단위로 구조안 2~3가지를 제시
- 각 구조의 장단점 + 추천안을 함께 보여줌

**6-2. 상세 플랜 작성** (사용자가 구조를 고른 후)
- 파트별 세부 내용, 시간/분량 배분
- 결정이 필요한 지점마다 선택지 + 추천안 제시
- 식상하거나 문제될 접근이 있으면 경고

**6-3. 최종 플랜 확정**

**왜 필요한가**: 같은 기준서로도 만드는 방법은 여러 가지입니다.
"페이지 1개에 다 넣을까, 탭으로 나눌까?" 같은 결정을 미리 하면 결과물이 훨씬 좋아집니다.
이걸 건너뛰면 AI가 자기 마음대로 구조를 정해버립니다.

---

### Phase 7: 만들기 + 점검 -- "복붙해서 만들고, 확인한다"

#### 7-1. Antigravity에서 만들기

1. Antigravity 열기 (구글 계정 로그인)
2. "New Project" -> 이름 입력 -> "Next.js" 선택 -> "Create"
3. AI 채팅에 **Phase 4에서 확정된 기준서 전체를 복사해서 붙여넣기**
4. "이 기준서대로 만들어줘" 라고 한다
5. 기다리기 (2~5분 정도 걸릴 수 있음)
6. "Preview" 탭에서 결과 확인

#### 7-2. 기준서와 비교하기

만들어진 결과를 보면서 기준서를 다시 펴세요.

```
기준서에 적힌 기능 목록과 지금 만들어진 앱을 비교해줘.
빠진 기능이 있으면 알려줘.
동작이 기준서와 다른 부분이 있으면 알려줘.
```

#### 7-3. 문제가 생기면 -- 원인부터 찾기

무작정 "고쳐줘"보다 **왜 문제인지** 먼저 파악하면 훨씬 잘 고쳐집니다.

```
[문제 설명].
이 문제의 원인이 뭔지 분석해줘.
가능한 원인을 목록으로 보여주고, 각각 가능성이 얼마나 되는지 알려줘.
```

| 증상 | AI에게 이렇게 말하세요 |
|------|----------------------|
| 계산 결과가 이상해 | "기준서의 공식이랑 실제 코드의 계산 로직을 비교해줘" |
| 디자인이 기준서와 다르게 나와 | "기준서의 디자인 방향과 지금 CSS를 비교해서 뭐가 다른지 보여줘" |
| 기능이 빠져있어 | "기준서의 기능 목록이랑 지금 구현된 기능을 대조해줘" |
| 에러가 나 | "이 에러 메시지의 원인이 뭐야? 가능한 원인 3가지 알려줘" |
| 모바일에서 깨져 | "모바일 화면에서 레이아웃이 어떻게 되는지 확인해줘" |
| 고쳤더니 다른 게 망가짐 | "이것만 고치는데 다른 건 건드리지 마" |

#### 7-4. 고쳤으면 기준서도 업데이트

```
방금 수정한 내용을 기준서에도 반영해줘.
기존 내용은 그대로 두고, 바뀐 부분만 추가/수정해줘.
```

다음에 비슷한 프로젝트를 할 때, 이 기준서를 가져다 쓸 수 있습니다.

## 좋은 기준서 vs 나쁜 기준서

| 나쁜 예 | 좋은 예 |
|---------|---------|
| "계산기 만들어줘" | "대출이자 계산기. 원리금균등/원금균등 선택, 월 상환액과 총 이자 표시" |
| "예쁘게 해줘" | "토스 느낌, 흰 배경, 파란 포인트 색상, 둥근 카드" |
| "계산 잘 해줘" | "이자 = 잔액 x 연이율 / 12, 원 미만 절사" |
| "모바일도 돼야 해" | "768px 이하에서 1열 레이아웃, 입력/결과 탭 전환" |
| (아무 제약 없음) | "서버/DB 없이 클라이언트만. 로그인 없음. 외부 API 없음" |
| "고쳐줘" | "기준서에는 이렇게 되어 있는데 실제로는 이렇게 동작해. 비교해줘" |

## 왜 이 순서인가

| 단계 | 하는 일 | 건너뛰면? |
|------|---------|----------|
| Phase 1 | AI와 인식 맞추기 | AI가 엉뚱한 걸 만듦 |
| Phase 2 | 체계적 정보 수집 | 기능이 빠지거나 쓸데없는 게 들어감 |
| Phase 3 | 빈틈+모순 점검 | 다 만들고 나서 "이거 빠졌네" 발견 |
| Phase 4 | 설계도 완성 | AI가 자기 마음대로 해석 |
| Phase 5 | 부족한 자료 확보 | 근거 없이 AI가 짐작으로 만듦 |
| Phase 6 | 구조 설계+플랜 | AI가 자기 마음대로 구조를 정함 |
| Phase 7 | 실행+점검+개선 | 문제를 발견 못하고 그대로 사용 |

---

## 문제가 생기면

| 상황 | 해결법 |
|------|--------|
| "Allow" 팝업이 뜸 | "Allow" 클릭. AI가 코드를 설치하는 정상 과정 |
| 빨간 에러 메시지 | 에러 메시지 전체를 복사해서 AI 채팅에 붙여넣기 |
| 화면이 멈춤 | 터미널에서 Ctrl+C -> AI에게 "다시 실행해줘" |
| 미리보기가 빈 화면 | AI에게 "미리보기가 안 돼, 확인해줘" |
| "Module not found" | AI에게 "외부 패키지 없이 다시 만들어줘" |
| 뭘 해야 할지 모르겠음 | AI에게 "지금 뭘 해야 돼?" |
| 전부 꼬였을 때 | 새 프로젝트 만들고 기준서 다시 복붙 (처음부터) |

핵심 원칙: **모르면 AI한테 물어보세요.** AI가 못 고치면 새로 시작하세요.

에러 대응 팁: "안 돼요"보다 **"이런 증상이 나와요"**가 더 정확한 답을 얻습니다.
화면 캡처나 에러 메시지 전체를 함께 주면 AI가 훨씬 정확하게 원인을 찾습니다.

# Part 2: AI용 실행 문서

> 아래 내용을 AI(ChatGPT, Claude, Gemini, Antigravity 등)에게 **그대로 복붙**하세요.
> 그 다음 "나는 ___을/를 만들고 싶어. 시작해줘." 라고 하면 됩니다.
> AI가 Phase 1부터 자동으로 진행합니다. 사용자는 답변, 확인, 선택만 하면 됩니다.

---

## AI 실행 규칙

- 아래 Phase를 순서대로 진행한다.
- 각 Phase가 끝나면 다음 Phase로 자동 이동한다.
- [사용자 입력]이 있는 곳에서만 멈추고 기다린다.
- 사용자가 답하지 않아도 AI가 할 수 있는 것(검색, 정리, 요약)은 AI가 직접 한다.
- 각 Phase의 산출물을 다음 Phase의 입력으로 사용한다.
- 기준서는 덮어쓰지 않는다. 수정 시 기존 내용을 보존하고 변경분만 추가한다.
- 매 Phase 끝에 AI가 이해한 내용을 요약하고, 사용자와 인식이 일치하는지 확인한다.
- 한번에 전체를 생성하지 않는다. 단계별로 생성 -> 검토 -> 정제를 반복한다.

---

## Phase 1: 프로젝트 파악

### AI 행동
사용자에게 질문한다:

```
무엇을 만들고 싶은지 자유롭게 설명해주세요.
(한 줄이든 한 페이지든 상관없습니다. 참고 자료가 있으면 같이 주세요.)
```

### [사용자 입력] 프로젝트 설명

### AI 행동
사용자 입력을 분석하여 정리한다:
- 프로젝트 유형 (웹사이트/앱/문서/기타)
- 추정되는 결과물 형태
- 언급된 자료 목록
- 아직 불명확한 부분 목록

정리 후 사용자에게 확인한다:
```
제가 이해한 내용이 맞는지 확인해주세요:
- 프로젝트: ___
- 결과물: ___
- 보유 자료: ___
- 불명확한 부분: ___

맞으면 "맞아", 수정할 부분이 있으면 말씀해주세요.
```

## Phase 2: 맞춤 질문지 생성 + 답변 수집

### AI 행동
Phase 1의 분석 결과를 기반으로, 이 프로젝트에 맞는 질문지를 생성한다.

질문지는 아래 5개 영역을 반드시 포함한다:

**A. 결과물 정의** -- 최종 형태, 성공 기준, 참고 서비스
**B. 대상 분석** -- 누가 쓰는지, 이미 아는 것, 원하는 것, 실망할 것
**C. 재료 점검** -- 가진 자료, 부족한 자료, 추가 수집 필요한 것
**D. 제약 조건** -- 시간, 예산, 필수 포함, 절대 금지
**E. 기존 사례** -- 비슷한 서비스가 어떻게 되어 있는지

각 질문에 반드시 붙일 것:
- [필요한 자료]: 이 질문에 답하려면 뭘 준비해야 하는지
- [찾는 법]: 그 자료를 어디서 어떻게 찾는지
- [답변 기준]: 답변이 충분한지 판단하는 기준

질문 수: 프로젝트 복잡도에 따라 12~18개

### [사용자 입력] 질문지 답변

사용자가 답하지 못하는 항목이 있으면:
- AI가 대신 검색/조사하여 초안을 작성한다
- 사용자에게 "이렇게 정리했는데 맞나요?"로 확인만 받는다

---

## Phase 3: 심화 분석 (2차 질문지)

### AI 행동
Phase 2의 답변을 분석하여 2차 질문지를 자동 생성한다.

2차 질문지에서 반드시 점검할 것:
1. 답변이 모호하거나 근거가 부족한 부분 -> 구체화 질문
2. 사용자 입장에서 "이게 정말 필요한가?" 검증
3. 서로 모순되는 부분 점검
4. 비슷한 서비스와 비교해서 부족한 점
5. 나중에 문제 될 수 있는 리스크
6. 결과물 구조/순서에 대한 대안 최소 2가지 비교
7. 놓치고 있을 수 있는 것

각 질문에 "왜 이 질문이 필요한지" 1줄 이유를 붙인다.

### [사용자 입력] 2차 답변

마찬가지로 AI가 조사 가능한 것은 AI가 먼저 하고 확인만 받는다.

2차 답변까지 완료 후 확인한다:
```
지금까지 파악한 프로젝트의 핵심입니다:
- 핵심 목표: ___
- 대상이 원하는 것: ___
- 가장 중요한 제약: ___
- 재료 분류: ___

이 이해가 맞으면 기준서 작성으로 넘어갑니다.
```

---

## Phase 4: 기준서 작성

### AI 행동
Phase 2 + Phase 3의 전체 답변을 바탕으로 기준서를 작성한다.

기준서에 반드시 포함할 항목:
- 프로젝트 기본 정보 (제목, 형태, 대상)
- 목표 (성공 기준과 연결)
- 대상 분석 요약 (아는 것 / 모르는 것 / 원하는 것 / 실망할 것)
- 필수 기능 목록 (우선순위 포함)
- 동작 규칙 (계산식, 처리 흐름)
- 화면 구조 (레이아웃)
- 디자인 방향 (스타일, 색상, 분위기)
- 제약 조건 (필수 / 금지)
- 확정된 결정사항 (Phase 2~3에서 확정된 것들)

### [사용자 입력] 기준서 검토 및 수정

사용자에게 기준서를 보여주고:
```
이 기준서를 검토해주세요.
수정할 부분이 있으면 말씀해주세요.
확정되면 "확정"이라고 해주세요.
```

확정될 때까지 수정 반복. 수정 시 기존 내용을 덮어쓰지 않고 변경분만 반영한다.

---

## Phase 5: 필요 자료 수집 + 정리

### AI 행동
기준서를 기반으로 자동 수행한다:

1. **자료 Gap 분석**: 기준서 달성에 필요한데 아직 없는 자료 목록
2. **수집 가능한 자료 직접 수집**: 웹 검색, 공식 문서, 사례 조사 등
3. **수집 불가능한 자료 요청**: 사용자만 가지고 있는 것 (개인 파일, 내부 자료 등)
4. **수집된 자료 요약 정리**: 기준서 항목별로 매핑

### [사용자 입력] (필요시) AI가 못 구하는 자료 제공

```
아래 자료는 제가 찾을 수 없어서 필요합니다:
- [자료 1]: 이유
- [자료 2]: 이유

나머지는 제가 수집 완료했습니다. 정리 결과를 확인해주세요.
```

---

## Phase 6: 구조 설계 + 플랜 생성

### AI 행동
기준서 + 수집된 자료를 바탕으로 실행 플랜을 작성한다.

**6-1. 전체 구조 먼저 제시**
- 큰 덩어리 단위의 구조안 2~3가지를 비교 제시
- 각 구조의 장단점 + 추천안

### [사용자 입력] 구조 선택

**6-2. 선택된 구조 기반으로 상세 플랜 작성**
- 파트별 세부 내용, 시간/분량 배분
- 결정이 필요한 지점마다 선택지 2~3개 + 추천안 + 추천 이유
- 식상하거나 누구나 하는 접근이 포함되어 있으면 지적 + 대안 제시
- "이대로 하면 안 되는 이유"가 있으면 반드시 경고

### [사용자 입력] 세부 선택

**6-3. 최종 플랜 확정**

플랜 출력 형식:
1. 전체 구조 요약
2. 파트별 상세 (확정된 선택 반영)
3. 필요 자료 체크리스트 (있는 것 / 없는 것)
4. 리스크 + 대안

---

## Phase 7: 실행 + 점검

### AI 행동
확정된 플랜에 따라 실행을 시작한다.
- 각 단계의 산출물을 만들면서 진행
- 기준서의 제약 조건을 매 단계에서 검증
- 완료된 단계는 체크하고 다음으로 이동
- 실행 중 발견한 문제나 개선점은 기준서에 변경분으로 반영

### [사용자 입력] 중간 확인 (필요시)
복잡한 산출물은 중간 확인을 받는다.

---

## Phase 요약

| Phase | AI가 하는 것 | 사용자가 하는 것 |
|-------|-------------|----------------|
| 1. 파악 | 분석, 정리, 확인 | 프로젝트 설명 |
| 2. 질문지 | 맞춤 질문 생성 + 자료 찾는 법 안내 | 답변 |
| 3. 심화 | 2차 질문 자동 생성 + 조사 | 답변 + 확인 |
| 4. 기준서 | 전체 답변을 기준서로 작성 | 검토 + 확정 |
| 5. 자료 | Gap 분석 + 수집 + 정리 | 못 구하는 자료만 제공 |
| 6. 플랜 | 구조안 비교 + 선택지 기반 플랜 | 선택 |
| 7. 실행 | 플랜대로 산출물 생성 + 기준서 업데이트 | 중간 확인 |
